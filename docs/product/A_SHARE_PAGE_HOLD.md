# A 股页面暂停决策与后续设计基线

状态：**暂缓实现**

决策日期：2026-07-18

恢复条件：产品负责人完成与其他开发者的数据源、展示权、缓存和署名边界讨论，并明确可进入生产的来源。

## 1. 暂停原因与当前边界

A 股页面仍在产品路线图中，但本轮不创建页面、路由、空数据表或无用缓存。原因不是技术不可行，而是拟议信源的公共访问方式与公开展示权没有同时得到确认：

- 东方财富存在可被网页调用的行情接口，但未找到面向本项目用途的正式公共 API 合同；其服务协议和法律声明明确提示，未经交易所书面同意不得复制、传播或散发行情数据。
- 雪球是信息聚合与社区产品，不是本项目可直接采用的数据再分发接口；其服务协议明确禁止未经授权的机器人、蜘蛛和爬虫方式，也限制第三方转载与平台内容再利用。
- “板块主力资金净流入”等字段通常包含供应商自定义算法，不等同于交易所原始事实。没有方法说明和展示许可时，不应把它包装成客观资金流。

因此，生产代码不得接入东方财富内部端点、雪球网页抓取、登录 cookie、逆向 header 或浏览器会话复用。免费访问不代表可以缓存或再分发。

## 2. 产品判断

四个指数适合作为风格与风险温度计，但不能单独代表整个 A 股市场：

| 指数 | 代码 | 页面角色 |
| --- | --- | --- |
| 中证 500 | `000905` / `399905` | 中盘与较高弹性风格 |
| 上证 50 | `000016` | 大盘蓝筹与权重风格 |
| 科创 50 | `000688` | 科技创新与高波动成长风格 |
| 创业板指 | `399006` | 成长风格与风险偏好 |

页面第一版应回答两个问题：大盘/风格今天在怎样变化，以及行业轮动是否在扩散或收缩。由于不做高频交易，首个 MVP 采用日终数据即可；只有在来源与额度允许时才增加低频盘中快照。

## 3. 恢复开发后的建议 MVP

### 3.1 指数区

四张指数卡统一显示：最新点位、当日涨跌幅、20 日实现波动、相对 20 日均线位置、观测时间和来源状态。页面必须区分“行情观测时间”和“SSE/SZSE 市场时钟状态”，不能用市场开盘状态推断行情已经刷新。

首版刷新建议：交易日收盘后一次（约 16:00–17:00）。如果之后确认盘中数据授权，可增加 `10:30`、`11:30`、`14:00`、收盘四个快照，不需要秒级轮询。

### 3.2 板块轮动区

优先做“轮动热度气泡图”，暂不命名为“资金流向图”：

- X 轴：板块相对基准的当日涨跌幅；
- Y 轴：成交热度，例如当前成交额相对过去 20 个交易日同时间中位数；
- 气泡面积：板块成交额占全市场成交额比例；
- 颜色：上涨/下跌方向，或相对前一快照的动量变化；
- 轨迹：若有低频盘中快照，显示板块在当天四个时点的移动路径。

这个设计只依赖价格与成交额即可描述轮动，不必把供应商专有的“主力净流入”算法当成事实。只有当来源明确给出可审核的资金流定义、字段方法和展示许可后，才增加真正的净流入维度。

## 4. 信源结论

| 候选来源 | 当前判断 | 恢复开发前必须确认 |
| --- | --- | --- |
| 交易所/指数公司官方数据 | 首选，但不自动等于可公开再分发 | API/下载方式、展示许可、缓存周期、署名、指数行情授权 |
| 东方财富网页与内部接口 | 当前禁止接入生产 | 正式 API 合同或书面授权，以及交易所行情展示权 |
| 雪球网页/聚合接口 | 当前禁止抓取或会话复用 | 正式开放 API、内容权利、缓存与公开展示授权 |
| Tushare 等文档化服务 | 可作为后续候选，不在本轮批准 | 具体套餐、非商业/公开网站边界、数据上游权利、再分发与署名 |
| Baostock 等开源客户端 | 仅列入调研，不在本轮批准 | 数据实际来源、许可证是否覆盖数据、更新稳定性与公开展示权 |

## 5. 已预留的架构契约

本轮通过文档和现有通用契约预留扩展点，不创建空实现：

- 未来 feature 边界：`app/src/features/a-share/`；
- 未来路由：`#/a-share`，恢复开发前不注册，避免出现空页面；
- 未来公共投影：`app/public/data/projections/a-share.json`，只由后端/CI adapter 生成；
- 未来指标命名空间：`a-share.index.*`、`a-share.sector.*`、`a-share.liquidity.*`、`a-share.northbound.*`；
- adapter 继续使用 fetch、normalize、validate、persist、project 五阶段和现有 `market_metric_observations` 事实表；
- SSE/SZSE 市场日历复用现有官方日历逻辑，但行情、板块数据与市场时钟分别保留 freshness/quality；
- 行业、指数、流动性、北向资金只预留分类名称，不提前创建表或存储未使用数据。

恢复开发时，先选定一个最小的官方/已审核来源并完成纵向 adapter，再注册页面和路由；不要先做 UI 后补来源合规。
