# Product Log

## 当前功能总览

- 加密周期地图：BTC、ETH、SOL、HYPE 月度收益矩阵、相对 BTC 收益、四年周期分组、读图结论、月内高低点顺序、固定详情卡和 URL 状态同步。
- 宏观流动性：通胀、就业增长、利率美元、波动信用、流动性和人工流动性事件日历，周历/月历会随本地日期自动定位。
- 美股大盘：QQQ、SPY、DIA 代理价格与 FRED 宏观数据的周度观察页，支持移动端 2x2 摘要、周度表格和可分享 hash 状态。
- 开市轮动：加密、美股、韩股和 A 股等市场的开闭市状态、样例行情、来源质量和轮动列表，移动端摘要改为 2x2。
- AI 芯片产业链：用细分类目和股票代码观察晶圆设备、材料基板、制造 IDM、存储 HBM、AI 芯片 IP、光模块互联、服务器、数据中心基础设施、软件终端应用和其他云算力需求的轮动。
- 机器人链：复刻机器人产业链观察池，按算力、感知、芯片、运动控制、自动驾驶、仓储服务机器人、医疗机器人、防务无人系统和 ETF 展示价格、涨幅和简要 K 线。
- 加密流动性：观察加密总市值、BTC 市值、主流稳定币与 USDT 脱锚幅度、BTC/ETH/SOL ETF 资金流，以及 Strategy 与 BitMine 的企业财库持仓和成本。
- 行情与缓存：前端只读静态 JSON；加密、市场时钟、美股大盘、宏观流动性、AI 芯片链和机器人链分别由本地/CI 脚本生成缓存，密钥不进入浏览器。

## 2026-07-13

- 加密流动性页新增企业财库需求区：展示 Strategy 的 BTC 与 BitMine 的 ETH 持仓、持仓成本、最新增持和现价相对成本；持仓日、成本日和披露日分别保存并展示，新闻稿中的现货价格不会被误当成采购均价。
- 企业财库优先采用 Strategy 官方披露与 SEC EDGAR 文件；BitMine 自动刷新仅在后端配置合规的 `SEC_USER_AGENT` 时启用，上游失败时保留已审核的 last-known-good 披露。
- SoSoValue 适配器升级到 Market Data API v1，预接 BTC、ETH、SOL ETF 与 Strategy 购买历史，并把 API 最近一个月窗口合并进既有历史；当前权限不足时 ETF 仍明确显示待接入，缺失交易日或空响应不会被写成零，也不会覆盖可用旧数据。BlockBeats 继续只作为默认关闭的辅助交叉检查源。
- 美股大盘新增日本 10 年期国债收益率：使用日本财务省官方 15:00 JST 收盘数据，并在摘要、日期详情和交互图表中保留独立的观察日期、来源与口径。
- 新增私有市场指标历史库及迁移文件：CI 在完整刷新后、发布前尝试把加密市值、稳定币、ETF、企业财库和日本利率的必要派生观测写入 Supabase；表启用 RLS、拒绝浏览器角色访问，并记录来源、观察/抓取/转换时间和修订审计。首次日本利率回填限制为最近 400 天，之后按短重叠窗口增量写入；数据库迁移完成前持久化失败不会阻断 Pages，完成后可用仓库变量 `MARKET_HISTORY_REQUIRED=1` 开启强制模式。
- 图表序列加入按来源 `fetchedAt` 合并的 last-known-good 保护，较旧的 provider 缓存不能反向覆盖较新的同日观测；Supabase 新式 opaque secret 仅作为 `apikey` 发送，旧式 service-role JWT 只保留迁移兼容。
- 公开市场缓存只保留可审计的来源状态与脱敏失败摘要，不再写入本机代理地址、请求参数或操作系统网络错误；日本国债抓取使用的 `requests` 也改为直接固定版本依赖。
